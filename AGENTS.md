## OVERVIEW

Monorepo for a client management system. Java 17 + Spring Boot 4 backend with Apache Wicket 7 UI, plus Angular 14 SPA frontend. PostgreSQL/MariaDB via Flyway migrations. MapStruct for DTO mapping, Lombok for boilerplate.


## CONVENTIONS

- **Backend commands**: `rtk gradlew compileJava` / `rtk gradlew test` / `rtk gradlew buildWar`
- **DB**: `docker-compose up -d mariadb` connects to `dummy_db`
- **Validation constraints**: Use `ValidationConstants.java` — no hardcoded limits in DTOs or validators
- **DTOs vs FormModels**: DTOs = immutable `record`s. FormModels = mutable beans for Wicket `CompoundPropertyModel`. Map on submit only.
- **Migrations**: Flyway naming — `V{number}__{description}.sql` in `db/migration/main/`
- **Mappers**: MapStruct in `com.desafio.estagio.mapper`
- **Angular API**: Generated by OpenAPI Generator from backend spec
- **bootRun with clean DB**: Use `docker compose -f docker-compose.test.yml up` — fresh MariaDB every time (`tmpfs`), API builds from `Dockerfile`
- **Wicket HTML**: Templates live alongside their Java component class in `src/main/java/`, not in resources

## ANTI-PATTERNS

- Wicket `<form>` tags inside `<table>/<tbody>/<tr>` — browsers hoist them, breaking AJAX DOM replacement. Bind `Form` to `<tr wicket:id="...">` instead.
- `@Transactional` at controller or view layer — must be at service layer only
- Overriding `AbstractClienteService.activate()`/`.inactivate()` — they're `final`
- Direct `EntityManager` usage — rely on Spring persistence context
- Creating per-type markup when shared component suffices — extract shared logic (e.g., `EnderecoCreateTablePanel`) but keep `wicket:id` type-specific
- `as any` / `@SuppressWarnings` — never suppress type errors

## COMMANDS

```bash
# Backend
rtk gradlew compileJava   # compile only
rtk gradlew test          # run tests
rtk gradlew buildWar      # build deployable WAR

# DB
docker-compose up -d mariadb  # start dev DB
docker compose -f docker-compose.test.yml up --build  # bootRun with clean DB
```
